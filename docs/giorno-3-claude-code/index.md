## 🎯 Cosa Imparerai

Nel Giorno 3 scoprirai **Claude Code**, la desktop app che porta le capacità di Claude direttamente sul tuo computer per task avanzati di documentazione, research e analysis.

Claude Code è potentissimo per sviluppatori, ma è anche incredibilmente utile per **professionisti non-tecnici**. Imparerai use case pratici per PM, PMO, Service Designers e Funzionali — zero coding richiesto.
:::

## 🛠️ Claude Code: Cos'è e Perché è Diverso

**Claude Code** vs **Claude Web (claude.ai)**:

| Feature | Claude Web | Claude Code |
|---------|------------|-------------|
| **Accesso file locali** | ❌ Upload manuale | ✅ Naviga directory |
| **Batch operations** | ❌ Uno alla volta | ✅ Multipli file |
| **Repository analysis** | ❌ Limitato | ✅ Analizza progetti interi |
| **Command execution** | ❌ No | ✅ Sì (read-only per non-dev) |
| **Documentation generation** | ⚠️ Manuale | ✅ Automatico |

**Quando usare Claude Code**:
- ✅ Analizzare molti documenti in una directory
- ✅ Generare/aggiornare documentazione automaticamente
- ✅ Estrarre insights da repository progetti
- ✅ Comparative analysis su dataset multipli
- ✅ Synthesis cross-document

**Quando usare Claude Web**:
- ✅ Quick questions e conversazioni
- ✅ File singoli da analizzare
- ✅ Brainstorming e ideation
- ✅ Mobile/tablet access

## 🔗 Anteprima Argomenti

<details>
<summary><strong>💻 Claude Code Setup & Overview</strong></summary>

Imparerai a:
- Installare Claude Code su tuo OS
- Navigare interfaccia e capabilities

- Differenze workflow vs Claude Web
- Best practices per file operations

**Focus non-dev**: Come usare Claude Code per task documentazione/analysis senza toccare codice.

</details>

<details>
<summary><strong>📚 Documentation Automation</strong></summary>

Imparerai a:
- Generare docs da file sorgente multipli
- Standardizzare formati e template
- Update automatico docs esistenti
- Consistency check cross-documents
- Export in vari formati

**Esempio**: Meeting notes sparse + emails + Slack exports → Project Charter strutturato con sections [Scope | Stakeholders | Timeline | Risks | Success Criteria].

</details>

<details>
<summary><strong>🔬 Research & Repository Analysis</strong></summary>

Imparerai a:
- Analizzare directory intere (non serve caricare file uno a uno!)
- Pattern recognition cross-files
- Comparative analysis strutturata
- Synthesis e executive summaries
- Data extraction da formati vari (PDF, CSV, DOCX, TXT)

**Esempio PMO**: Directory con 30 status reports → analizza automaticamente → genera dashboard trend (on-time%, budget variance, top risks).

</details>

<details>
<summary><strong>🎓 Progetto Finale Integrativo</strong></summary>

- Usa prompt engineering (Giorno 1)

- **PM**: Onboard nuovo team member → genera onboarding doc da project artifacts

- **Service Design**: User research synthesis → 15 interviste → themes + insights + recommendations
- **Funzionale**: Requirements traceability → AS-IS docs → TO-BE specs → gap analysis

</details>
